# Cache Dump — mapped blocks

Human-readable view of every parameter record in the 4 main effect
blocks, extracted from AM4-Edit's metadata cache. Generated by
`scripts/gen-cache-enums.ts` — do not hand-edit. Regenerate after
any change to the cache by running:

```
npx tsx scripts/parse-cache.ts
npx tsx scripts/gen-cache-enums.ts
```

**Wire protocol:** cache record `id` == wire `pidHigh`. The block's
`pidLow` is in the section header below. Send SET_PARAM with
`(pidLow, pidHigh=id)` and the value encoded per the unit convention
in `src/protocol/params.ts`. Type-dropdown id is 10 for most blocks
except Compressor (id=19) and GEQ (id=20).

Parameter *names* are not in the cache — only IDs. Entries whose
name is known via wire capture are tagged in `params.ts`. Everything
else here is "cache-derived, unnamed" and needs either an AM4-Edit
capture or a manual review to label.
